# Odisena Console ‚Äî Netlify CI/CD

Automated GitHub Actions + Netlify CLI deploy pipeline for the Odisena
Console PWA. This repo is a **static, no-build** site, so `build_command` is
empty and `publish_dir` is `"."` (the repo root is uploaded as-is).

## What the pipeline does

1. **Production deploys** ‚Äî pushes to `main` run `netlify deploy --prod`
   via the official Netlify CLI (`npx netlify-cli`). There is no build step.
2. **Branch & PR previews** ‚Äî pushes to any non-`main` branch, and pull
   requests, create a Netlify preview deploy (`netlify deploy --alias`) and
   post the preview URL to the GitHub Actions step summary and (for PRs) a
   PR comment.
3. **HTTPS / custom-domain readiness** ‚Äî after a production deploy, the
   workflow polls `https://console.odisena.com` a few times and reports
   whether DNS/SSL are ready. It **never edits DNS** and never forces
   certificate issuance ‚Äî Netlify auto-issues a Let's Encrypt certificate
   only after DNS is correctly pointed at Netlify. A not-ready domain only
   prints a warning (it does not fail the run ‚Äî see
   `domain_check.fail_workflow_on_not_ready` in `odisena-deploy.yml`).
4. **Slack notifications** ‚Äî a message is posted to the Odisena deployments
   Slack channel after each deploy attempt (repo, branch, commit, deploy
   URL, environment, status).

## Files

| File | Purpose |
| --- | --- |
| `.github/workflows/odisena-netlify-deploy.yml` | The GitHub Actions workflow. |
| `odisena-deploy.yml` | Per-repo config (site name, build/publish, custom domain, Slack). **No secrets.** |
| `scripts/odisena-netlify-deploy.mjs` | Reads `odisena-deploy.yml`, deploys via Netlify CLI, checks domain readiness. |
| `scripts/odisena-slack-notify.mjs` | Posts the Slack deploy notification. |
| `netlify.toml` | Real Netlify config for this repo (headers/cache). Already present; not generated by the template. |
| `netlify.toml.example` | Reference showing production/deploy-preview/branch-deploy contexts. |
| `.github/dependabot.yml` | Keeps GitHub Actions versions current (monthly). |

## Required GitHub secrets

Add under **Settings ‚Üí Secrets and variables ‚Üí Actions**:

**Required**

| Secret | Value |
| --- | --- |
| `NETLIFY_AUTH_TOKEN` | Netlify personal/team access token (User settings ‚Üí Applications ‚Üí Personal access tokens). |
| `NETLIFY_SITE_ID` | Netlify Site ID / API ID for the Console site (Site configuration ‚Üí General ‚Üí Site details). |
| `SLACK_WEBHOOK_URL` | Slack Incoming Webhook URL for the Odisena deployments channel. |

**Optional**

| Secret | Purpose |
| --- | --- |
| `NETLIFY_TEAM_SLUG` | Overrides `team_slug` (`i-w-dansereau`); used for dashboard links only. |
| `NETLIFY_CUSTOM_DOMAIN` | Overrides `custom_domain` (`console.odisena.com`) without editing the committed file. |
| `ODISENA_DEPLOY_CONFIG_PATH` | Path to `odisena-deploy.yml` if not at the repo root. |
| `SLACK_BOT_TOKEN` + `SLACK_CHANNEL_ID` | Alternative to `SLACK_WEBHOOK_URL` (Slack app/bot with `chat:write`). |

The workflow fails fast with a clear error if `NETLIFY_AUTH_TOKEN` or
`NETLIFY_SITE_ID` is missing. Missing Slack credentials only warn ‚Äî Slack
failures never fail the deploy.

## Custom domain (manual, outside GitHub)

The workflow **checks** readiness only. To make `console.odisena.com` live:

1. Netlify: **Site configuration ‚Üí Domain management ‚Üí Add a domain** ‚Üí
   `console.odisena.com`.
2. In DNS (GoDaddy): add the record(s) Netlify specifies (typically a
   `CNAME` for the subdomain pointing at `<site>.netlify.app`). **Do not
   change `odisena.com` or `www.odisena.com` records** ‚Äî those host the
   public brand site.
3. Wait for propagation; Netlify auto-provisions the Let's Encrypt cert once
   DNS resolves to Netlify.
4. The next `main` deploy's readiness check reports success once
   `https://console.odisena.com` responds.

## Safety notes

- No credentials live in any committed file. `odisena-deploy.yml`,
  `netlify.toml`, and `netlify.toml.example` hold non-sensitive config only.
- The workflow **never modifies DNS**; the domain check is a read-only HTTPS
  GET.
- Preview deploys never publish to the production URL or custom domain ‚Äî only
  `main` pushes do.

See the source template at `odisena-netlify-cicd-template` for the full
reference documentation and build-based-repo variations.
