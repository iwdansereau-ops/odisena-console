Choosing a runner for regression benchmarks

Wall-clock distribution — violin + box

Each violin is 200 back-to-back runs. Wider = more common. Box = IQR. Dot = median.

Coefficient of variation

σ / μ × 100. Lower CV = tighter, more repeatable timings.

Jitter range (max − min)

Peak-to-peak wall-clock spread across the run cohort.

Run-by-run wall-clock timeline

Each point is one run. Spikes reveal noisy neighbors, cgroup throttling, GC pauses.

Rolling median (window = 10 runs)

Smoothed to expose systemic drift vs. one-off outliers.

Instructions per run should be near-deterministic. ASLR + memory layout + JIT stochasticity are the usual noise sources — hypervisors amplify all three.

Retired instructions — mean ± σ (via cachegrind)

QEMU's user-mode translation multiplies retired instructions ~6×. Bare-metal is the reference.

L1 D-cache miss variance

Run-to-run CV of L1 data-cache misses. Higher = more cache disturbance from co-tenants.

Context switches per run

Direct signal of scheduler pressure. Bare-metal < 30, QEMU > 500.

Methodology

Each of five CI environments executed three CPU-intensive workloads 200 times back-to-back on a pinned core with Turbo Boost disabled. Wall-clock time is clock_gettime(CLOCK_MONOTONIC). Retired instructions and L1 D-cache misses come from a separate run under valgrind --tool=cachegrind, which is deterministic modulo ASLR and layout stochasticity.

The composite score weighs three noise-floor dimensions:

  • Wall-clock CV (50%) — the primary target for benchmark regression detection.
  • Instruction CV (35%) — how deterministic the retired-instruction count is. Any drift here means the runner cannot be trusted for micro-regression detection.
  • Throughput (15%) — median wall time. Cheap noise is still noise, but slow runners burn CI budget.

Warm-up (first 3 runs) is excluded from statistics; it is retained in the timeline view so cold-start effects remain visible.

How to read the scorecard

A regression benchmark is only useful if its noise floor is smaller than the effect you are trying to detect. If your target regression is a 3% wall-clock slowdown, you need CV well below 1% — anything above and false positives will drown the signal.

QEMU user-mode ARM emulation appears here as a cautionary tale: retired instruction counts inflate ~6× from translation overhead, and their variance jumps by more than an order of magnitude. Cross-arch benchmarks belong on native hardware.

Docker containers with tight cgroup CPU quotas produce a signature failure mode — clean-looking medians hiding periodic tail spikes as the quota window resets. Watch p99 more than mean.
